SCP and SSH Without Entering a Password

Apr 15, 2016

Both scp and ssh prompt you for a password when trying to connect to a remote host. Here is how to do both without having to enter a password.

Assume that localhost means the machine running the scp/ssh command and remotehost means the remote machine you are connecting/transferring files to.

If you don’t already have a pair of private/public keys, generate a new set with ssh-keygen. Don’t forget to set valid permissions if you are creating a new key pair.

This will create a private key ~/.ssh/id_rsa and public key ~/.ssh/id_rsa.pub pair on your localhost.
