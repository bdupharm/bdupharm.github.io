Using Forked NPM Modules

Jun 27, 2016

You just forked an npm module you’ve been using for a while and you’re presented with two options.

  1. Wait for the maintainer of that module to merge your pull request
  2. Use the forked version in the meantime

Ehhh, only yeezus knows when this guy is going to merge our PR… so let’s just use our forked version for the time being.

In most cases, using a forked module will be very easy. Just add the git url to package.json. See the NPM documentation here.

BUT WAIT, what if the project’s author has a clever way of publishing his module to NPM?

If you take a look at this React.JS module and peer into the gulpfile.js and .npmignore files, you can see that this guy is not merely running npm publish. The author threw some custom build/publish code inside of a gulp task.

To solve this, we ended up just running the build step, tarballing the result and saving the tarball within the project and editingpackage.json like so.

{
  "dependencies": {
    "boron": "./some-folder/boron-fork.tar.gz"
  }
}

Alternatively, you could host the tarball somewhere like AWS S3 and add in the url in package.json.
